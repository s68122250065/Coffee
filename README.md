# Coffee
Phase 1: คำถามอภิปราย

1.ทำไมลำดับการเสร็จไม่เหมือนเดิมทุกครั้ง?
ตอบ แต่ละ Thread ทำงานพร้อมกัน และระบบปฏิบัติการเป็นตัวจัดลำดับการทำงานของ Thread ทำให้บางครั้งเมนูหนึ่งอาจเสร็จก่อนหรือหลังต่างกันได้ ลำดับจึงไม่เหมือนเดิมทุกครั้งที่รันโปรแกรม

2.ถ้าเรียก run() แทน start() จะเกิดอะไรขึ้น?
ตอบ จะไม่เกิด Thread ใหม่โปรแกรมจะทำงานเหมือนเรียกเมธอดปกติทำให้ชงกาแฟทีละเมนูตามลำดับไม่ได้ทำพร้อมกัน

3.main thread ทำงานพร้อมกับ CoffeeMaker หรือไม่?
ตอบ ทำงานพร้อมกันเพราะเมื่อใช้ start() จะสร้างThreadใหม่ให้ CoffeeMaker ในขณะที่ main thread ยังทำงานต่อได้เลย เช่น พิมพ์ "รับออเดอร์ครบแล้ว" โดยไม่ต้องรอการชงกาแฟให้เสร็จก่อน



Phase 2: แบบฝึกหัดจากกรณีศึกษา
1. เพิ่มจำนวนเมนูเป็น 10 รายการ แล้วสังเกตชื่อ thread
ตอบ เมื่อเพิ่มเมนูเป็น 10 รายการ โปรแกรมจะสร้าง Thread หลายตัวและการทำงานของแต่ละ Thread จะสลับกัน ไม่ได้ทำตามลำดับเมนูเสมอ

2. เปลี่ยน Thread.sleep(2000) เป็นสุ่ม Math.random()*3000 แล้วดูความสลับของ output
ตอบ เมื่อเปลี่ยนเวลาเป็นแบบสุ่ม แต่ละเมนูจะใช้เวลาชงไม่เท่ากัน ทำให้ลำดับการเสร็จของเครื่องดื่มสลับกันมากขึ้น

3. ทดลองเรียก run() แทน start() แล้วอธิบายผล
ตอบ ถ้าใช้ run() โปรแกรมจะไม่สร้าง Thread ใหม่ทุกคำสั่งจะทำใน main thread ทีละเมนูทำให้ผลลัพธ์เรียงตามลำดับไม่ได้ทำงานพร้อมกัน

4. ใส่ System.out.println("End main") ต่อท้าย แล้วดูว่า main จบก่อนชงเสร็จหรือไม่
ตอบ บางครั้ง "End main" จะแสดงก่อนที่เครื่องดื่มจะชงเสร็จ เพราะ main thread ไม่ได้รอ Thread อื่นให้ทำงานเสร็จ

5. แก้ปัญหาโดยใช้ synchronized และ AtomicInteger
ตอบ ใช้ synchronized เพื่อให้ Thread ใช้เครื่องชงกาแฟได้ทีละตัวและใช้ AtomicInteger เพื่อเพิ่มค่าตัวนับออเดอร์ได้อย่างปลอดภัยเมื่อมีหลาย Thread ทำงานพร้อมกัน
